# Polymarket 15 分鐘自動交易 Bot — 策略報告

> 寫給有基本程式能力但沒有量化交易經驗的人。

---

## 一、我們在做什麼？

Polymarket 是一個預測市場平台。它每 15 分鐘開一個新市場：

> 「BTC 在接下來 15 分鐘會漲還是跌？」

你可以買「漲」或「跌」的股份（share）。如果你押對了，每股值 $1；押錯了，變 $0。

**重點：股份的價格不是固定的。** 價格反映市場共識：
- 「漲」的股份賣 $0.80 → 市場認為 80% 機率會漲
- 「漲」的股份賣 $0.10 → 市場認為只有 10% 機率會漲

我們的 bot 做的事情：**在市場共識過度偏向一邊時，買入被低估的那一邊。**

---

## 二、為什麼這能賺錢？（邊際優勢來源）

### 人會恐慌，機器不會

想像 BTC 在過去 5 分鐘跌了 $200。大部分人（和他們的 bot）會認為「還會繼續跌」，瘋狂買「跌」的股份。這時「漲」的股份變得很便宜（比如 $0.10）。

但數據告訴我們：**BTC 在 15 分鐘內的漲跌接近隨機**，跌了之後不一定繼續跌。如果「漲」的實際機率是 40%（不是市場定價的 10%），那：

```
買 1 股「漲」花 $0.10
  如果漲了 → 拿 $1.00（淨賺 $0.90）
  如果跌了 → 損失 $0.10

期望值 = 40% × $0.90 - 60% × $0.10 = $0.30（正的！）
```

**賺錢不是因為我們能預測方向，而是因為市場定價太極端時，即使猜錯的次數多，猜對的利潤也能覆蓋虧損。**

### 手續費的秘密

Polymarket 對「Taker」收 ~1.56% 手續費（在 50/50 賠率時）。但費用公式是：

```
fee = price × 0.25 × (price × (1 - price))²
```

當 price = $0.50 → fee = 1.56%（很貴）
當 price = $0.10 → fee = 0.02%（幾乎免費）
當 price = $0.05 → fee = 0.005%（基本為零）

**我們只在極端賠率（便宜的那邊）下注，所以幾乎不付手續費。**

更好的是：如果用 **Limit Order（掛單）** 而不是 Market Order（吃單），**手續費直接是 0%**，還能拿到 20% 的回扣。

## 三、具體策略是什麼？

我們的 bot 實現了一個叫「Krajekis 5 層框架」的策略：

### 第 1 層：看時段

不同時間段的市場行為不同：

| 時段 | 特徵 | 我們的做法 |
|------|------|-----------|
| 亞洲（UTC 0-8 時） | 安靜、窄幅震盪 | 做均值回歸（價格偏離就反向押） |
| 倫敦（UTC 7-10 時） | 殺止損、假突破 | 等假突破結束再反向入場 |
| 紐約（UTC 12-16 時） | 最大波動、真突破 | 跟隨趨勢方向 |
| 週末 | 流動性差 | 小倉位，只做極端回歸 |

### 第 2 層：看波動率

用 ATR（Average True Range）判斷目前市場有多「瘋狂」：

| 波動率 | BTC 15分鐘 ATR | 策略 |
|--------|---------------|------|
| 低（平靜） | < $80 | 買「貴的那邊」（$0.70-0.95），方向大概率不變 |
| 高（暴動） | > $200 | 買「便宜的那邊」（$0.02-0.25），賭它回來 |

### 第 3 層：看技術指標

Bot 從 Binance 即時拉取 1 分鐘 K 線，計算：
- **RSI**（動量）：> 70 = 超買（可能回落），< 30 = 超賣（可能反彈）
- **MACD**（趨勢加速度）：柱狀圖擴大 = 趨勢加速，縮小 = 趨勢衰竭
- **EMA 21/50**（趨勢方向）：快線在慢線上 = 多頭
- **VWAP**（成交量加權均價）：價格偏離 VWAP 越遠 → 回歸壓力越大

### 第 4 層：看窗口時機

每 15 分鐘有一個新市場。我們只在市場開始後 5-10 分鐘進場（「甜點區」）：
- 太早（前 5 分鐘）：噪音太大，方向不明
- 太晚（最後 2 分鐘）：價格已經反映，沒有 edge

### 第 5 層：風險管理

- 每筆 1 USDC（起始資金的 ~10%）
- 每天最多虧 3 筆就停
- 最多同時持有 2 個倉位

## 五、兩種賺錢場景

### 場景 A：低波動時的趨勢跟隨（穩定小賺）

```
情況：BTC 在平靜時段，價格在 VWAP 上方，EMA21 > EMA50
     → 趨勢向上，15 分鐘內大概率不會翻轉

市場定價：Up = $0.75, Down = $0.25
我們買：Up @ $0.75

結果 A（漲了，80% 機率）：拿 $1.00，淨賺 $0.25
結果 B（跌了，20% 機率）：損失 $0.75

期望值 = 80% × $0.25 - 20% × $0.75 = +$0.05
```

**利潤小（每筆 5-10 cents），但頻率高、勝率高。**

### 場景 B：高波動時的均值回歸（偶爾暴擊）

```
情況：BTC 剛跌了 $300，RSI = 18（極度超賣），MACD 開始收縮
     → 市場恐慌，但統計上反彈機率不低

市場定價：Up = $0.08, Down = $0.92
我們買：Up @ $0.08

結果 A（漲了，35% 機率）：拿 $1.00，淨賺 $0.92 = 11.5:1 回報！
結果 B（跌了，65% 機率）：損失 $0.08

期望值 = 35% × $0.92 - 65% × $0.08 = +$0.27
```

**利潤大（暴擊），頻率低，但一次抵多次虧損。**

## 七、風險

### 這不是穩賺的

1. **我們不知道真實勝率** — 51% 的 WR 是朋友的回測數據，不是我們的。我們的 TA 信號可能更差。

2. **Polymarket 費用在變** — 3/30 起 crypto fee 從 1.56% 漲到 1.80%（50/50 時）。如果用 Maker 下單則不受影響（0%）。

3. **流動性風險** — 15 分鐘市場的 liquidity 只有 ~$40K。大單會滑價，但我們只下 $1 所以不受影響。

4. **Oracle 風險** — Polymarket 用 Chainlink 預言機結算。如果 Chainlink 延遲或出錯，結算可能不準。

5. **監管風險** — Polymarket 在美國受限。我們用日本 IP（Oracle Cloud 大阪），目前可以交易，但政策可能改變。

6. **過擬合風險** — krajekis 策略沒有統計驗證。我們用的 TA 指標（RSI/MACD/EMA）在 15 分鐘級別的預測力可能接近零。**真正的 edge 可能不是 TA 信號，而是 Polymarket 的定價低效。**

### 最壞情況

起始資金 $9.88 USDC，每筆 $1。如果連續 9 筆全虧（機率極低但可能），損失全部資金。

**但這正是用小資金測試的意義 — 用 $10 驗證策略，而不是用 $10,000。**

## 八、下一步

1. **觀察 dry run 24 小時** — 確認 dedup 和新 ATR threshold 正常運作
2. **切 real mode** — 用真金白銀跑 1 週
3. **記錄每筆交易的結算結果** — 計算真實勝率
4. **如果 WR > 50% 且正期望值** → 逐步加倉（$5 → $10 → $50）
5. **如果 WR < 48%** → 停止，分析哪個層出了問題

## 九、與之前 TSMOM 策略的根本差異

| | 舊 TSMOM 策略 | 新 Polymarket 策略 |
|---|---|---|
| **做什麼** | 預測 BTC 未來 7 天方向 | 在 Polymarket 押 15 分鐘方向 |
| **怎麼賺** | 趨勢持續時賺差價 | 市場定價偏差時賺賠率 |
| **Edge 來源** | IC ≈ 0.02（幾乎是噪音） | 群眾恐慌 + 費用結構不對稱 |
| **在場時間** | 80%（幾乎永遠持倉） | ~10%（只在甜點區進場） |
| **成本** | Funding Rate 吃掉大部分利潤 | Maker 下單 0% 手續費 |
| **最大虧損** | 帳戶餘額的 100%（有槓桿） | 每筆固定 $1（無槓桿） |
| **回測 vs 實盤** | 回測 SR=3.85，實盤全虧 | 還沒有回測，直接小額實測 |
